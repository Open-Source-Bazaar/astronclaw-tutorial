# 实战：打造你的个人 AI 搭子

很多人对 AI 助手的期待，已经从「问一句答一句」变成了「一个越用越懂我的伙伴」——它记得我的偏好、能跨设备接续上次的对话，还能基于我的习惯主动帮我把事情往前推。

好消息是，你不需要自己搭一套复杂的记忆系统。**AstronClaw（云端大脑）+ Loomy（桌面入口）** 这套组合已经把所需的能力都准备好了，本篇带你把它们拼成一个真正属于你的「个人 AI 搭子」。

## 它由哪些部分组成

| 角色 | 承担什么 | 对应能力 |
| :--- | :--- | :--- |
| **AstronClaw（云端）** | 7×24 小时在线的大脑，托管模型与技能，电脑关机也不掉线 | [核心优势与对比](/guide/astronclaw/introduction)、[技能系统](/guide/astronclaw/skills) |
| **Loomy（桌面）** | 本地入口，连接你的文件、应用与即时通讯渠道 | [介绍与定位](/guide/loomy/introduction) |
| **AI 搭子** | 有性格、懂你节奏的协作人格 | [AI 搭子](/guide/loomy/companions) |
| **资料库** | 让搭子能安全访问并记住你本机的资料 | [资料库](/guide/loomy/knowledge-base) |
| **定时任务** | 让搭子在固定时间主动替你执行流程 | [定时任务](/guide/loomy/scheduled-tasks) |
| **远程控制** | 通过 QQ / 微信 / 飞书 / 钉钉跨设备指挥 | [远程控制](/guide/loomy/remote-control) |

下面分三步，把这些能力组装成一个「记得你、跟得上你、还会主动帮你」的搭子。

## 第一步：让搭子「记得你」

「记忆」不是某一个开关，而是由三层叠加出来的效果。逐层配置好，搭子才会越用越像你的人。

### 1. 用「搭子」固化协作风格

AI 搭子会在一次次协作里慢慢熟悉你的节奏。先选一个贴合你工作方式的搭子，或直接创建一个专属的：

> 帮我创建一个「日常工作搭子」：定位是我的个人助理，说话简洁直接、结果导向，熟悉我做的是科技内容运营，回复时默认用要点列表。

这样每次对话都不必从头交代背景，搭子的人设本身就是一份「关于你」的长期设定。创建与调整方法见 [AI 搭子](/guide/loomy/companions)。

### 2. 用「资料库」沉淀你的资料

把常用目录授权给资料库后，Loomy 会在**本地**持续扫描并自动归类你的文档与图片——这相当于给搭子一份随时可查的「个人知识库」。

> 你在资料库里帮我找一下上个月那份项目复盘文档，结合里面的结论，帮我起草这个月的计划。

这样搭子的回答会建立在你真实的历史资料之上，而不是泛泛而谈。

> **隐私说明**：资料的扫描、归类和索引都在你的本地设备完成，内容不会上传。详见[资料库](/guide/loomy/knowledge-base)。

### 3. 用「自定义模型」掌控成本与延续性

如果你已经有稳定的模型服务，可以在 Loomy 中配置自己的 API Key。使用自有 Key 时模型调用通常不消耗 Loomy 积分，长期高频使用更划算。配置方式与积分规则见 [定时任务与积分说明](/guide/loomy/scheduled-tasks)。

> **API Key 安全**：你配置的 Key 只保存在本地设备，Loomy 不会托管或上传。

## 第二步：跨设备接续

个人搭子的价值在于「人走了，活还在干」。这一步靠 AstronClaw 的云端常驻 + Loomy 的远程控制来实现。

- **云端不掉线**：AstronClaw 在云端 7×24 小时运行，不依赖你的电脑是否开机，这是「随时找得到」的基础。
- **消息渠道遥控**：给 Loomy 接入 QQ / 微信 / 飞书 / 钉钉机器人，你在手机上发一句话就能触发任务、查看进度、取回结果。接入步骤见 [远程控制](/guide/loomy/remote-control)。

典型用法：上午在工位用桌面端布置一个整理任务，中午离开后在手机微信里问一句「上午那个整理任务跑完了吗？把结果发我」，搭子就会把结果回传过来。

> **前提**：远程控制和定时任务都依赖 Loomy 在本地设备**保持后台运行**。如果应用被完全退出，任务将无法触发。

## 第三步：让搭子主动帮你——「每日简报与计划官」

把前两步攒下的「记忆」用起来，配合定时任务，就能得到一个会主动工作的搭子。下面是一个可直接照搬的例子。

**目标**：每天早上 8:00，让搭子结合你的资料库和昨日进度，生成一份当天的简报与计划，并通过远程渠道推送给你。

**定时任务 Prompt 示例**：

> 每天早上 8:00 执行：
> 1. 汇总今天合肥的天气，以及科技领域值得关注的 3 条热点；
> 2. 在资料库里翻一下我最近的「周计划」文档，列出今天应该推进的任务；
> 3. 扫一眼未读邮件，标出需要今天回复的；
> 4. 把以上整理成一张「今日简报」卡片，并通过微信机器人发给我。

它把多项能力串成了一条个性化流程：

* **主动性**：定时触发，不用你每天手动开口。
* **记忆**：从资料库读取你真实的计划与历史。
* **个性化**：搭子按你设定的人设与偏好来组织输出。
* **跨设备**：结果直接推送到手机，离开工位也收得到。

> **稳定执行建议**：涉及多步操作或外部工具的任务，建议先**手动跑通一次**，确认权限和工具都就绪，再设为定时任务，可显著降低自动执行时的跑偏概率。

## 进阶建议

* **一人多搭子**：可以按场景养不同的搭子——稳健推进型用于工作复盘，创意陪伴型用于内容发散。切换搭子就是切换协作风格。
* **从手动到自动**：先在日常对话里把一个流程打磨顺，再固化成定时任务，成功率最高。
* **善用搜索弹框**：用快捷键随时唤起[搜索弹框](/guide/loomy/knowledge-base#搜索弹框)，在任意界面把任务直接交给搭子。

## 常见问题

**搭子真的会「记住」我说过的话吗？**
搭子的「记忆」来自三处：人设设定（固化你的协作偏好）、资料库（你授权的本地资料）、以及任务与对话上下文。把前两者配置好，就能稳定地获得「越用越懂你」的体验，而不依赖某个单独的记忆开关。

**资料库会把我的文件传到云端吗？**
不会。授权后，扫描、归类与索引都在本地完成，内容仅用于本地检索。

**离开电脑后还能用吗？**
可以查看与触发。AstronClaw 云端常驻保证「找得到」，但真正落到本地的任务执行仍需 Loomy 在设备上后台运行。
